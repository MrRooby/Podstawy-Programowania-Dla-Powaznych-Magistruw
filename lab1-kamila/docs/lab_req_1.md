Define the ModelARX class interface (public methods) according to the following guidelines.   The class should provide:    -   Setters for coefficient vectors for both polynomials.  

Tips:        Note that polynomial setter s should also define the sizes of the signal buffers. Hints:      Value constraints must be implemented manually in the setter body (as value - c hecking code).   Simply defining the delay setter argument as unsigned does not solve the problem, as a   negative value can still be entered. Tips:        Defining an argument constructor aut omatically excludes the default constructor, but   it's good programming practice to explicitly exclude it with the delete specifier. An   argument constructor should definitely use defined setters to avoid unnecessary code   duplication  (DRY  rule ) .  Don't  set  field  values    directly,  or  in  the  constructor's   initialization list, as this not only circumvents constraints but also bypasses the   execution of important code responsible for adjusting buffer sizes.       Stick with the YAGNI rule to avoid  defining anything unnecessary. Therefore, there's no   need to define copy and move constructors, let alone their corresponding assignment   operators, because at this stage the class doesn't have any external dependencies that   require redefining their default   versions (created automatically by the compiler).   There's also no need to define getters for fields. If such a need arises in the future, they   can always be added.       4.
